# Contributing

Contributions are welcome! To get started:

1. Fork the repository.
2. Create a branch for your changes: `git checkout -b feature/my-new-feature`.
3. Make your changes and commit them: `git commit -m 'Add a new feature'`.
4. Push your changes: `git push origin feature/my-new-feature`.
5. Open a pull request.

## AI-Assisted Development

The use of AI is permitted in this project, but it must be accompanied by total transparency. We do not judge the use of these tools, but we strictly oppose "contribution farming"—the practice of submitting unverified or uncomprehended code solely to increase PR counts.

If you use AI to assist with your contribution, you are required to disclose its use in your Pull Request description, specifying:
- **When** it was used (e.g., for boilerplate, for complex logic, or for research).
- **How** it was used (e.g., as a discussion partner, for documentation, or to explore a crate).
- **Why** it was used (e.g., to overcome a language barrier like Rust, or to navigate complex D-Bus interactions).

By submitting a contribution, you commit to the following:

1. **Logical Ownership**: You must fully understand every line of code you submit. The core logic must be yours, not a "black-box" generated by an LLM.
2. **Responsibility for Correctness**: You are solely responsible for ensuring that your changes do not introduce any regressions—whether they be functional bugs, crashes, or visual glitches.
3. **Source Primacy**: AI output must always be verified against official documentation or source code to ensure technical accuracy and adherence to project standards.
4. **Respecting Intellectual Lineage**: If your contribution is inspired by a specific developer, project, or implementation pattern, please acknowledge them in your PR description. We honor the work that flows into this project.
